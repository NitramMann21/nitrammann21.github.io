Contributing

Do you want something related to the Twitch API to be in the library? Do you have issues using the JTA? No problem, contributing is as easy as creating an issue / pull request and voila, you're contributing to the API! At least if it is something related to the library.

Creating an issue

1.: Go to the issues tab of the repository (or click here).
2.: Click on NEW ISSUE, choose whether it is a bug report or a feature request, fill in the title and description and click on CREATE ISSUE.
That should have created your issue! Please wait until someone replies to your issue.

Creating a pull request

Creating a pull request is a bit trickier.
1.: Create a fork. To do this, click the FORK button on the top right of the screen while on the repository's main page. Please only fork from development.
2.: Pull the repository to your local machine. Either use GitHub Desktop, the terminal or your IDE.
3.: Add the differences you want to make. If your change is not internal, please add documentation.
4.: To create a pull request, go to the main page of your repository, click on Contribute and then on Open pull request. Then click on Create pull request, fill in the title and the description and click on Create pull request one final time.
5.: After creating the request you have to wait for the developers to approve the changes. If you want to change your changes, you can always commit to your forked repository. If the request was approved, you can delete your fork. DO NOT DO THIS UNTIL THE PULL REQUEST WAS EITHER MERGED OR CLOSED!
